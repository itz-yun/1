## 1. SRS Wall Kick 資料表

**決策**：使用 The Tetris Company 官方 SRS Wall Kick 偏移資料表（2009 標準）

**J、L、S、T、Z 方塊 Wall Kick 偏移表**：

```javascript
// 格式：[dx, dy]（正 x = 右，正 y = 下）
const WALL_KICK_JLSTZ = {
  '0→R': [[ 0, 0], [-1, 0], [-1, 1], [ 0,-2], [-1,-2]],
  'R→2': [[ 0, 0], [ 1, 0], [ 1,-1], [ 0, 2], [ 1, 2]],
  '2→L': [[ 0, 0], [ 1, 0], [ 1, 1], [ 0,-2], [ 1,-2]],
  'L→0': [[ 0, 0], [-1, 0], [-1,-1], [ 0, 2], [-1, 2]],
  // 逆時針（反向）
  'R→0': [[ 0, 0], [ 1, 0], [ 1,-1], [ 0, 2], [ 1, 2]],
  '2→R': [[ 0, 0], [-1, 0], [-1, 1], [ 0,-2], [-1,-2]],
  'L→2': [[ 0, 0], [ 1, 0], [ 1, 1], [ 0,-2], [ 1,-2]],
  '0→L': [[ 0, 0], [-1, 0], [-1,-1], [ 0, 2], [-1, 2]]
};

// I 方塊專用（獨立資料表）
const WALL_KICK_I = {
  '0→R': [[ 0, 0], [-2, 0], [ 1, 0], [-2,-1], [ 1, 2]],
  'R→2': [[ 0, 0], [-1, 0], [ 2, 0], [-1, 2], [ 2,-1]],
  '2→L': [[ 0, 0], [ 2, 0], [-1, 0], [ 2, 1], [-1,-2]],
  'L→0': [[ 0, 0], [ 1, 0], [-2, 0], [ 1,-2], [-2, 1]],
  'R→0': [[ 0, 0], [ 2, 0], [-1, 0], [ 2, 1], [-1,-2]],
  '2→R': [[ 0, 0], [ 1, 0], [-2, 0], [ 1,-2], [-2, 1]],
  'L→2': [[ 0, 0], [-2, 0], [ 1, 0], [-2,-1], [ 1, 2]],
  '0→L': [[ 0, 0], [-1, 0], [ 2, 0], [-1, 2], [ 2,-1]]
};
```

**演算法**：按順序測試每個偏移，若套用後無碰撞則旋轉成功；若全部失敗則取消旋轉。

**理由**：SRS 是現代競技標準（TETR.IO、Puyo Puyo Tetris），可防止方塊卡住並提升遊戲體驗。  
**考慮的替代方案**：NES 原版（無 Wall Kick，已過時）、ARS 旋轉系統（不同資料表，非主流）

## 2. 7-Bag 隨機系統

**決策**：依 Tetris Guideline 實作 7-bag 隨機系統（Fisher-Yates 洗牌）

**演算法**：

```javascript
class SevenBagRandomizer {
  constructor() {
    this.PIECES = ['I', 'O', 'T', 'S', 'Z', 'J', 'L'];
    this.currentBag = this._shuffle([...this.PIECES]);
    this.nextBag    = this._shuffle([...this.PIECES]);
    this.index = 0;
  }

  next() {
    if (this.index >= 7) {
      this.currentBag = this.nextBag;
      this.nextBag    = this._shuffle([...this.PIECES]);
      this.index = 0;
    }
    return this.currentBag[this.index++];
  }

  peek() {
    // 預覽下一個方塊（不消耗）
    return this.index < 7
      ? this.currentBag[this.index]
      : this.nextBag[0];
  }

  _shuffle(arr) {
    for (let i = arr.length - 1; i > 0; i--) {
      const j = Math.floor(Math.random() * (i + 1));
      [arr[i], arr[j]] = [arr[j], arr[i]];
    }
    return arr;
  }
}
```

**理由**：保證每 7 個方塊出現一次各類型，消除連續出現相同方塊的挫敗感。  
**考慮的替代方案**：純隨機（可能連續出現 8+ 個同類型，體驗差）

## 4. 計分系統

**決策**：使用 Tetris Guideline 官方計分規則

| 事件 | 分數 |
|------|------|
| 消除 1 行 | 100 × 等級 |
| 消除 2 行 | 300 × 等級 |
| 消除 3 行 | 500 × 等級 |
| 消除 4 行（Tetris） | 800 × 等級 |
| Soft Drop（每格） | 1 分 |
| Hard Drop（每格） | 2 分 |

**理由**：官方標準確保計分結果可預測，Soft/Hard Drop 積分鼓勵主動操作。  
**考慮的替代方案**：NES 經典計分（公式不同，已過時）

## 5. 遊戲迴圈架構

**決策**：`requestAnimationFrame` + 獨立重力計時器（deltaTime 累積）

```
每幀（約 16.67ms，60fps）
├─ 輸入處理（即時消耗 keyState/touchState）
├─ 重力更新（gravityTimer += deltaTime；達到 interval 才下移）
└─ 渲染（clearRect → 繪製 Board → 繪製方塊 → 繪製 Ghost → 繪製 UI）
```

**理由**：重力計時與輸入分離，確保不同幀率下遊戲速度一致；deltaTime 累積可容忍掉幀。  
**考慮的替代方案**：`setTimeout`（可能與渲染脫節）；每幀固定下移（依賴幀率）

## 6. 渲染方案

**決策**：HTML5 Canvas（單一 `<canvas>` 元素）

| 面向 | Canvas | DOM（div 格子） |
|------|--------|---------------|
| 200 格/幀渲染 | ~0.2ms | ~5-8ms |
| 60fps 穩定性 | ✅ 穩定 | ⚠️ 可能卡頓 |
| 記憶體 | 單一點陣圖緩衝 | 200+ DOM 節點 |
| 實作複雜度 | 中等 | 簡單 |

**Canvas 規格**：
- 主遊戲區域：`300px × 600px`（10 欄 × 30px，20 列 × 30px）
- 預覽區：`120px × 120px`（4 欄 × 30px，4 列 × 30px）
- `imageSmoothingEnabled = false`（像素完美渲染）

**理由**：Tetris 每幀需更新最多 200 格，Canvas 渲染速度是 DOM 的 5-10 倍。  
**考慮的替代方案**：DOM divs（較慢）、WebGL（過度複雜）、SVG（向量，不適合格子遊戲）

## 7. 觸控手勢偵測

**決策**：自訂滑動偵測（閾值法），不使用第三方函式庫

**閾值**：

| 參數 | 值 | 說明 |
|------|-----|------|
| 最小滑動距離 | 30px | 防止誤觸 |
| 最長滑動時間 | 300ms | 超過視為拖曳 |
| 點擊最大位移 | 10px | 在此範圍內視為點擊 |

**手勢對應**：

| 手勢 | 操作 |
|------|------|
| 左滑 | 向左移動方塊 |
| 右滑 | 向右移動方塊 |
| 下滑 | Soft Drop（加速下落） |
| 上滑 | Hard Drop（立即落底） |
| 單點（Tap） | 旋轉方塊（順時針） |

**理由**：30px 閾值為行業標準，可過濾手指微小位移；不依賴 Hammer.js 保持零依賴。  
**考慮的替代方案**：Hammer.js（增加 7KB 依賴，超出需求）

## 8. 速度進展公式

**決策**：指數型加速，每等級提升 10%（`800ms / 1.1^(等級-1)`，最低 50ms）

| 等級 | 間隔（ms） | 速度（格/秒） |
|------|-----------|--------------|
| 1  | 800  | 1.25 |
| 2  | 727  | 1.38 |
| 3  | 661  | 1.51 |
| 5  | 547  | 1.83 |
| 10 | 343  | 2.92 |
| 15 | 217  | 4.61 |
| 20 | 108  | 9.26 |
| 29+| 50   | 20.00（上限）|

等級計算：`level = Math.floor(totalLinesCleared / 10) + 1`

**理由**：指數增長提供平滑的難度曲線；50ms 上限防止難度失控。符合規格 SC-005 要求（等級 10 約 2.6 格/秒）。  
**考慮的替代方案**：線性遞減（後期太慢）、NES 固定對應表（不一致）

## 9. localStorage 最高分

**決策**：使用 `localStorage.getItem/setItem('tetrisHighScore', score)`

```javascript
const HIGH_SCORE_KEY = 'tetrisHighScore';

function getHighScore() {
  return parseInt(localStorage.getItem(HIGH_SCORE_KEY) || '0', 10);
}

function saveHighScore(score) {
  const current = getHighScore();
  if (score > current) {
    localStorage.setItem(HIGH_SCORE_KEY, score.toString());
    return true; // 更新成功
  }
  return false;
}
```

**理由**：localStorage 是瀏覽器原生 API，無需任何依賴；JSON 序列化對單一數值來說過度複雜，直接存字串即可。
